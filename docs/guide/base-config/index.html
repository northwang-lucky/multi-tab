# 基础配置

从这里开始，将详细介绍如何使用本项目。

# 路由与侧边栏

侧边栏中的各个菜单项是与路由模块一一对应的，增添或修改菜单项等同于增添或修改路由。

这里给出路由meta中可配置的选项：

{
  // 可选项：'item', 'submenu', 'group'，分别代表：普通菜单项，下拉菜单项，分组菜单项
  type: 'item',
  // 显示在菜单上的名称
  label: '标题',
  // svg图标名，值为src/assets/icons/svg目录下svg文件的文件名
  icon: 'title',
  // 是否将该路由显示在侧边栏中
  hidden: false,
  // 哪些权限有资格访问该路由，值为string或string[]
  roles: 'common',
  // 是否将该路由对应的菜单项设置为禁用态
  disabled: false,
  /**
   * 该路由对应菜单项的点击事件
   * @param event 点击事件的MouseEvent
   * @param item  一个包含该菜单项全部属性的对象
   * 设置该点击事件后，不会触发路由跳转
   */
  onClick: (event, item) => {}
}

# 一种添加页面的场景

举个栗子，假设你已经写好了add-user.vue (添加用户的页面)，也下载好了添加用户的 svg 小图标 add-user.svg，现在想要添加一个用户管理的菜单项，下面包含添加用户的子级菜单项：

在src/router/modules目录下，新建一个user.ts文件：

import { RouteRecordRaw } from 'vue-router'
import AppMain from '@/layout/components/AppMain'

const userRoutes: RouteRecordRaw = {
  name: 'user',
  path: 'user',
  component: AppMain,
  redirect: '/user/list',
  meta: {
    order: 3, // 控制菜单项的排序
    type: 'submenu', // 可选项：'item', 'submenu', 'group'，分别代表：普通菜单项，下拉菜单项，分组菜单项
    label: '用户管理', // 显示在菜单上的名称
    icon: 'menu' // svg图标名
  },
  children: [
    {
      name: 'addUser',
      path: 'add',
      component: () => import('@/views/user/add-user'),
      meta: {
        type: 'item',
        label: '添加用户',
        icon: 'add-user'
      }
    }
  ]
}

export default userRoutes

保存之后，菜单项就会自动更新了。

# 另外一种场景

举个栗子，假设我已经写好了一个visual.vue (可视化的页面)，并且下载好了对应的小图标 visual.svg，想要将这个页面添加到工作台这个菜单项的下面，作为一个一级普通菜单项：

打开src/router/routes.ts，找到工作台 (workbench) 对应的路由，并在其下方输入可视化 (visual) 的路由配置：

{
  name: 'workbench',
  path: 'workbench',
  component: () => import('@/views/workbench'),
  meta: {
    type: 'item',
    label: '工作台',
    icon: 'workbench'
  }
},
{
  name: 'visual',
  path: 'visual',
  component: () => import('@/views/visual'),
  meta: {
    type: 'item',
    label: '可视化',
    icon: 'visual'
  }
}

保存之后，菜单项就会自动更新了。

# 菜单项作为超链接的场景

举个栗子，公司有一个老项目，需要从新项目中超链接跳转：

{
  name: 'oldSystem',
  path: 'oldSystem',
  component: {},
  meta: {
    type: 'item',
    label: '旧管理系统',
    icon: 'old-system',
    onClick: () => window.open('https://example.xxx.com/old', '_blank')
  }
}

给路由设置 onClick 属性之后，不会触发项目内的路由跳转，而是会执行 onClick 中的内容。

# vuex 模块

本项目对 vuex 进行了模块化拆分。

# 新增模块

打开src/store/typings/index.d.ts，新增一个模块类型：

export interface RootStateProps {
  userModule: UserStateProps
}

export interface UserStateProps {
  isLogin: boolean
  userId: string | number
  username: string
  token: string
}

在src/store/modules目录下，新建user.ts文件：

import { Module } from 'vuex'
import { UserStateProps, RootStateProps } from '../typings'

const userModule: Module<UserStateProps, RootStateProps> = {
  namespaced: true,
  state: {
    isLogin: false,
    userId: '',
    username: '',
    token: ''
  },
  mutations: {
    SET_LOGIN(state, status: boolean) {
      state.isLogin = status
    },
    SET_USER_ID(state, userId: string | number) {
      state.userId = userId
    },
    SET_USERNAME(state, username: string) {
      state.username = username
    },
    SET_TOKEN(state, token: string) {
      state.token = token
    }
  }
}

export default userModule

PS：当启用namespace属性时，mutation与action的调用需要加上前缀，例如，调用SET_LOGIN时应该这样写：

store.commit('userModule/SET_LOGIN', true)

# 模块数据持久化

项目中使用了vuex-persist进行模块数据持久化处理，例如，将user模块中的数据保存在localStorage中只需要在src/store/index.ts中这样写：

import VuexPersist from 'vuex-persist'

const vuexLocal = new VuexPersist({
  storage: window.localStorage,
  key: 'multi-tabs-local',
  modules: ['userModule']
})

const store = createStore<RootStateProps>({
  plugins: [vuexLocal.plugin]
})
